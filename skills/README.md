# Skills

Skills are grouped by capability, not by SDLC phase.

Canonical skills will live under this directory after proposal validation,
promotion, and registry admission. Current capability folders are placeholders
for future canonical skill definitions.

## Skill Organization

Capability folders:

- `strategy/`
- `ontology/`
- `requirements/`
- `ux/`
- `architecture/`
- `execution/`
- `devex/`
- `governance/`
- `bootstrapping/`

Skills should remain agent-agnostic and reusable. Workflows and Process Trees
decide sequencing.

## Canonical Skill Discovery

The Canonical Skill Registry is the authoritative inventory for approved,
deprecated, archived, and proposed skills.

Use the registry to discover:

- skill ID
- capability
- lifecycle state
- repository location
- Process Tree nodes
- current version
- source proposal
- dependencies

## Relationship to the Skill Registry

`skills/` stores canonical skill assets.

The Skill Registry indexes skill assets and tracks lifecycle, placement,
Process Tree relationships, dependencies, and traceability.

Canonical skills must appear in the registry. Deprecated skills must remain
traceable. Archived skills must remain discoverable through registry references.

Do not add full skill implementations until the relevant proposal, validation,
promotion, lifecycle, and registry requirements are satisfied.
