# Proposals

Proposals are the official intake zone for framework evolution.

New expertise, assets, and structural changes must enter ai-driven-sdlc as
proposals before they can become canonical framework assets.

Approved assets remain stable. Proposals allow the framework to review,
validate, and approve changes without modifying canonical skills, workflows,
Process Trees, schemas, adapters, or installation assets directly.

## Proposal Categories

- `skills/`: skill proposals
- `workflows/`: workflow proposals
- `process-trees/`: Process Tree proposals
- `schemas/`: schema proposals
- `adapters/`: adapter proposals
- `installers/`: installation asset proposals

## Rules

- Do not place canonical assets in `proposals/`.
- Do not promote a proposal without validation and approval.
- Preserve source prompts and protocols according to Prompt Immutability.
- Record proposal lifecycle state, ownership, validation evidence, and promotion
  readiness.
