# Workflows

Workflows compose capability skills into SDLC sequences.

This directory does not yet contain full workflow definitions. Future workflows
should remain readable, portable, and agent-agnostic.

## Rules

- Workflows orchestrate skills into phase-like sequences.
- Workflows should not duplicate reusable skill content.
- Workflow inputs, outputs, and decision points should be explicit.
- Avoid depending on a specific AI vendor, editor, or CLI.

## Candidate Future Workflows

- raw concept to structured requirements
- requirements to architecture
- architecture to execution plan
- governed implementation review
